# 🍳 レシピブック (GitHub Pages)

Markdownファイルで手軽に管理できる、自分だけのレシピブックサイトです。
GitHubにMarkdownをプッシュすると、GitHub Actionsが自動でビルドし、GitHub Pagesに最新のレシピサイトを公開します。

---

## 🌟 特徴
- **自動ホームページ生成**: `recipes/` にMarkdownを追加するだけで自動構築。
- **複数ジャンル機能**: 各レシピにカンマ区切りで複数のジャンルを登録でき、一覧ページで自動的にジャンル分けされます。
- **人数変更 & 材料自動計算**: 1〜20人前まで選択可能。人数に応じて材料の分量が自動で比例計算されます（分数表記にも対応）。
- **アコーディオン手順**: 作り方は初期状態で全開ですが、調理が終わった手順から「完了」ボタンを押して折りたたんでいくことができます。
- **いいね機能**: よく作る料理に「いいね」を押して、一覧ページの上位にソートできます（データはブラウザの `localStorage` に保存されます）。
- **ダークモード対応**: OSのテーマ設定に連動して自動でスタイリッシュなダークカラーに切り替わります。
- **モバイルファースト**: キッチンでスマホを見ながら料理しやすい、見やすく操作しやすいデザイン。


## 📝 レシピの追加方法

### 1. テンプレートをコピーする
ルートディレクトリにある `recipe_template.md` をコピーし、`recipes/` フォルダの中に **半角英数字のファイル名**（例: `curry.md`, `hamburg-steak.md`）で配置します。

### 2. レシピ内容を記述する
コピーしたファイルをテキストエディタで開き、内容を書き換えます。

#### 💡 フロントマター（基本情報）の書き方
ファイルの冒頭部分にレシピ情報を記述します。
```yaml
---
title: "カレーライス"
genre: "洋食, 定番料理, スパイス"  # カンマ(,)区切りで複数ジャンルを設定可能！
description: "じっくり煮込んだコクのある定番カレー。"
servings: 4                     # 基準人数（材料の分量の基準）
updated: "2026-06-21"           # 更新日 (YYYY-MM-DD)
---
```

#### 💡 材料の書き方（自動計算対応）
`## 材料` セクションに、`- [材料名]: [分量]` の形式で書きます。
- **自動計算させたい場合**: `200g` や `1/2個`、`大さじ1` のように**数値を最初（または単位の前）**に書いてください。
- **自動計算させない場合**: `適量` や `少々` など数値を含めずに書くと、人数を変えてもそのまま表示されます。

#### 💡 手順とポイントの書き方
`## 手順` セクションに、`### [工程タイトル]` で工程を区切ります。
- 各工程のすぐ下に説明文を書きます。
- `> **ポイント**: コツや注意点` の形式で書くと、レシピページで「💡 ポイント」として見やすく装飾されます。

## 🖼️ 画像の掲載方法

画像は **`recipes/img/`** フォルダに入れます。画像が配置されていればページに自動で反映されます。画像がない場合はデフォルトの料理アイコン（🍽️）が表示され、エラーにはなりません。

| 種類 | 画像ファイル名のルール | 例（レシピが `curry.md` の場合） |
|---|---|---|
| **完成形写真**（一番上と一覧のサムネイル） | `[mdファイル名]_complete.jpg` （※） | `curry_complete.jpg` |
| **手順写真**（アコーディオン手順の中に表示） | `[mdファイル名]_step[手順の連番].jpg` | `curry_step1.jpg` <br> `curry_step2.jpg` |

> ※ 一覧ページのサムネイルには、`[mdファイル名]_thumb.jpg` という画像がある場合のみ表示されます。サムネイル用画像がない場合は、一覧にはプレースホルダーが表示され、個別レシピページのみに完成形写真が表示されるため、動作が重くなりません。
> ※ 対応拡張子: `.jpg`, `.jpeg`, `.png`, `.webp` （ビルドスクリプトが自動で探します）

## 💻 ローカルでの動作確認手順

新しく追加したレシピを自分のパソコンでビルドして確認する方法です。

### 1. 依存関係のインストール（初回のみ）
```bash
npm install
```

### 2. ビルドを実行
```bash
npm run build
```
ビルドが完了すると、`docs/` フォルダの中に公開用のHTMLファイル群が生成されます。

### 3. ローカルサーバーを起動
```bash
npm run dev
```
起動後、ブラウザで `http://localhost:3000` にアクセスして動作確認を行います。

## 🚀 GitHubへの公開・更新手順

ファイルを編集・追加したら、変更内容をGitHubにプッシュするだけで完了です。

```bash
git add .
git commit -m "Add new recipe: curry"
git push origin main
```
プッシュすると、GitHub Actionsが自動的に立ち上がり、数分後に最新のレシピサイトがインターネット上に公開されます！
（進捗はGitHubの「Actions」タブから確認できます）
